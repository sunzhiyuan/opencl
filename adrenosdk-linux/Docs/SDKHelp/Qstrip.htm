Qstrip

Qstrip is a simple library for the application level optimization of vertex index data. The resulting index information may be used in calls to glDrawElements.

Qstrip comes with two header files exposing two data structs:

  • Qsplit - takes sets of triangle indices as input and return multiple triangle sets with a restricted number of vertices indexed by each contiguous set of triangles
  • Qstrip - takes indexed triangles as input (3 indices per triangle) and return a single index list of triangle strips representing a vertex compression of the previous index data. The strip may be optimized for degenerates or for cache coherency to make the functionality more useful on a broader range of platforms.

The objective of the Qsplit functionality is to split complex sets of triangle data into smaller contiguous subsets that will remain resident on DSP data FIFOs during the transformation and lighting process.

The objective of Qstrip is to minimize the vertex transformation and lighting overhead by sharing vertices between triangles through indexed triangle strip reuse and reissuing those indices where possible while they are still resident in hardware vertex cache.

For more information about Qstrip, please consult the documentation included in the tool's directory.
