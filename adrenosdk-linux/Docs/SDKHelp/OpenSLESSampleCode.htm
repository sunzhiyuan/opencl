OpenSL ES™ Samples

The VS2010 sample code provided in the SDK for OpenSL ES™ covers 4 basic topics that are the fundemental building blocks of creating a full-featured middleware audio API.

Wrapping the OpenSLES.dll (provided with the sample code) is sample_framework.c/h, which simplifies the use of OpenSL ES™ calls to a set of standardized functions.

samples.c provides the main code that shows how to use the API. int main() contains 4 return functions, and each function gives an overview of a common requirement found in most audio engines today.

SimpleSample()

SimpleSample loads a sample into memory and plays it. The sound is looped and will play for one second before it fades out. When the fade is complete, the sample will be stopped. This is a good example of how to load an audio sample into an application and play it. Useful for playing 2D U.I. type sounds.

Sample3D()

Sample3D loads a sample and plays it positionally in a circle around a listener position (at 0,0,0). This is a good example of how to make 3D audio content in an application.

SampleEffects()

SampleEffects loads a sample and assigns it to be played with virtualization on (i.e. reverberation). This is a good example of how one might place procedural effects on a sound within an application to add realism and detail to a sound.

SampleStream()

SampleStream opens a file handle to a sound and plays the sound in via a buffering system. This is a good example of how one might want to play audio that is too large to fit into memory (i.e. music, ambient noises, dialog etc.).
