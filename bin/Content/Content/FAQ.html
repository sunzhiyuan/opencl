Frequently Asked Questions

My mobile device is plugged in, and my application is running. Why is nothing listed in Adreno Profiler's connection box?

• Confirm USB drivers have been installed for your device according to the instructions in Device Installation.

• Before an Android application may be profiled, it must have the android.permission.INTERNET permission enabled in its manifest.

• Before profiling, and after rebooting your Android device, you must enable debug mode by setting the following ADB property on the command-line:

adb.exe shell setprop debug.egl.profiler <1 to enable, 0 to disable>

On 2.x versions of Adreno Profiler this step is done for you automatically. Simply launch the Adreno Profiler connect dialog before launching your application.

• For Windows Mobile devices, Adreno Profiler must be running, and ActiveSync connected, before starting the embedded graphics application.

• Make sure the embedded application creates a hardware-accelerated OpenGLES context, and is not falling back to a software reference driver.

• If ActiveSync or Android Debug Bridge crashes or closes abnormally, Adreno Profiler may be unable to scan the device after it has been reconnected. Close and restart Adreno Profiler.

• If you use firewall software, ensure that its exception list includes Adreno Profiler (AdrenoProfiler.exe).

- Additionally, if using a Windows Mobile device, ensure all ActiveSync processes and services are also listed as firewall exceptions: rapimgr.exe, wcescomm.exe, WCESMgr.exe, and port 26675 for the ActiveSync Service.

- If using an Android or BREW-based device, no additional firewall exceptions are needed.

- If the above suggestions do not resolve your connection problem, please contact Technical Support.

The Scrubber is not rendering my captured frame correctly.

If a captured frame does not render identically in both Adreno Profiler and the device, please let us know.

When profiling a DirectX application, why does the profiler crash with an error about missing dll or relevant depedencies?

If you get the error, "System.IO.FileNotFoundException: Could not load file or assembly 'DXPlaybackWrapper.dll' or one of its dependencies.
The specified module could not be found. File name: 'DXPlaybackWrapper.dll'", this is most likely due to DirectX runtime not being installed on your machine.
Please install DirectX Runtime from this location.
For any other crashes, please contact us.

Before contacting Technical Support, however, please consult the list of known issues.
