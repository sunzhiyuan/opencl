Adreno Profiler
Scrubber GL :: Emulator View

The Scrubber GL interface encapsulates a set of powerful tools for capturing and analyzing rendered frames from the connected graphics application.

To open a new Scrubber GL, click on its button on the main application toolbar.



Scrubber metrics usage:

• Please see the tutorial on capturing per-rendercall metrics.



Mouse controls:

• To pan the scene, drag with the left mouse button.

• To pick a mesh in the scene, click on it with the left mouse button; the call trace will then seek to the selected mesh.

• To seek to the end of the frame, click anywhere in the black border around the scene.



Toolbar options:

Capture a new frame of animation from the device.

• Upon a capture, Adreno Profiler will download the OpenGLES call trace of the mobile application’s next frame of animation, along with associated VBO, vertex attribute, uniform, shader, and program data. The frame is then rendered within Adreno Profiler through an OpenGLES emulation environment.

• If the texture browser’s “Sync On Capture” option is enabled, the embedded driver’s current texture store will also be downloaded and applied to the scene.

• Note that any edits made to shaders from a previously captured frame will be destroyed upon a new capture.


Choose which framebuffer surface to display: color, depth, or stencil.


View the current device FBO.

• If toggled on, the currently bound framebuffer object, as of the selected draw call, is displayed in the emulator view.

• If toggled off, the primary EGL surface will always be displayed, even if the device has bound a framebuffer object.


Display the current device binning configuration.

• When enabled, the emulator view will overlay a grid of lines representing the device's framebuffer bin configuration.

• The number of bins required for your rendering surfaces has a big effect on performance. Always use the smallest framebuffer configuration possible; for example, experiment with a 2-byte color format, less or no bits of depth and/or stencil, and various MSAA settings.


Scene visualization modes.

[] Normal: the scene is rendered as it is on the device

[] Wireframe: the scene is rendered in wireframe mode


The next three modes portray overdraw in the scene. The brighter a pixel is, the more fragments at that screen location have passed the depth test and been rasterized. This mode is useful for analyzing the effectiveness of depth-sorting algorithms in game engines.

[] Show Overdraw: leaves the depth test state unchanged from what was set by the mobile application

[] Show Overdraw: forces depth testing off

[] Show Overdraw: highlights any pixels rejected by the depth test for the captured scene; the more brightly colored a pixel is, the more fragments at that screen location have been rejected by the depth test during the rendering of the scene


Rotate the emulator view clockwise.


Zoom the viewport to actual size.


Zoom the viewport to fit the bounds of the emulator window.
