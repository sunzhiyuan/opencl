Adreno Profiler
OpenCL Profiling Metrics Available

All of the metrics below are provided for each executed kernel which is being profiled.

  • Total Kernel Cycles - Count of GPU clock cycles to execute the kernel.

  • % Kernel Load Cycles - Percentage of cycles used for kernel loading. This is the amount of GPU execution time spent outside the Shader Processor.

  • % GPU busy - Percentage of time the GPU is busy. Higher is better.

  • % GPU Idle - Percentage of time the GPU is idle. Lower is better.

  • % ALU Utilization - Percentage of time that the ALU is busy.

  • % Load-Store Busy - Percentage of time the Load-Store unit is busy.

  • % Write L2 Hit - Percentage of write L2 cache hit from all SPs. Higher is better. Negative numbers indicate high cache trashing.

  • % Read L2 Hit - Percentage of read L2 cache hit from all SPs. Higher is better. Negative numbers indicate high cache trashing.

  • % L1 Hit - Percentage of L1 cache hit. Higher the better. Negative numbers indicate high cache trashing.

  • % L1 Misses - Percentage of L1 cache misses. Lower is better.

  • L1 Hit/Miss Ratio - L1 Hit/Miss ratio for all SPs. Higher is better.

  • # Full ALU - Number of full precision ALU instructions executed for the kernel profiled.

  • # Half ALU - Number of half precision ALU instructions executed for the kernel profiled.

  • # EFU - Number of Elementary function instructions executed for the kernel profiled.

  • # Local Loads - Number of local memory load instructions.

  • # Local Stores - Number of local memory store instructions.

  • # Local Atomics - Number of local memory atomic instructions.

  • # Global Loads - Number of global memory load instructions.

  • # Global Stores - Number of global memory store instructions.

  • # Global Atomics - Number of global memory atomic instructions.

  • # Global Read Data - Total amount of data read from Global memory in Bytes.

  • # Global Write Data - Total amount of data written into Global memory in Bytes.

  • % Texture Busy - % L1 cache usage when compared to Shader Processor.
