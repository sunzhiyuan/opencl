Adreno Profiler
OpenCL Profiling Metrics Available

The following statistics are provided for any kernel that is compiled in the kernel analyzer:

  • All Instructions - Total number of assembly level instructions that the kernel got compiled into.

  • ALUs - Number of ALU instructions.

  • Half ALUs - Number of half ALU instructions.

  • NOPs - Number of No Operation instructions. These indicate the fillers used for providing synchronizations between the various ALU’s

  • EFU Instrs - Number of EFU instructions.

  • MOVs - Number of data move instructions.

  • FP converts - Number of floating point conversions.

  • Loads/Stores - Number of global load store assembly instructions. The lesser these are in comparison to ALUs/Half ALU instructions the better.

  • Flow Instrs - Number of flow instructions.

  • Texture Instrs - Number of instructions using the texture pipeline.

  • Sync Instrs - Number of sync instructions.

  • Full Registers - Number of full registers used by the kernel

  • Half Registers - : Number of half registers used by the kernel. The number of full and half registers directly determine the number of waves that will be created for the kernel. The lesser the register usage, the higher the number of waves created.

  • Number of Threads - Number of threads or waves of the kernel. Indicates how many waves are created for the kernel. Higher the better. Max is 16 waves. The lesser the number of waves, the lesser the data latency can be hidden.
