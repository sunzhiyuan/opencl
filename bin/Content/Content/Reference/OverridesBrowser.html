Adreno Profiler
Overrides Browser

The Overrides Browser lists the GLES context overrides supported for the connected application.





Note: Overrides are disabled until a Scrubber document is created.



Basic usage:

• Double-click an override to toggle it on or off.

• Experiment with overrides while monitoring performance to try to isolate performance bottlenecks in the connected application. Examples:

- Use "DisableGL" to determine whether or not the application is graphics-bound. If disabling GLES API calls causes a large increase in framerate, it implies the application is graphics bound; if the framerate improvement is small, application code is the limiting factor.

- Use "ForceScissorTestReject" to determine whether or not the application is fillrate-bound. Enabling the override forces a 1-by-1 scissor window to be enabled, which leaves per-vertex computation unchanged but prevents most per-fragment work from taking place. If the override triggers a large increase in framerate, it implies the application is bound by fragment operations.
