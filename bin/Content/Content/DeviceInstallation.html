Device Installation

Android Device Setup

For Android devices, the Android Debug Bridge (ADB) must be installed in order to use Adreno Profiler. ADB should be downloaded as part of the latest Android SDK release from Google, which may be obtained from the following link:

Android SDK

After downloading and unpacking the Android SDK, add the path location of adb.exe and AdbWinApi.dll to your system PATH environment variable. Ensure that there are no versions of ADB in the profiler directory and no additional versions of ADB are accessible via the PATH environment variable.

An appropriate USB driver must also be installed for your Android device. A Windows USB driver for the HTC G1 may be obtained here:

http://dl.google.com/android/android_usb_windows.zip

Additionally, before an Android application may be profiled, it must have the android.permission.INTERNET and com.qti.permission.PROFILER permissions enabled in its manifest.

Finally, before profiling, and after rebooting your Android device, you must enable debug mode by setting the following ADB property on the command-line:

adb.exe shell setprop debug.egl.profiler <1 to enable, 0 to disable>

On 2.x versions of Adreno Profiler this step is done for you automatically. Simply launch the Adreno Profiler connect dialog before launching your application.

Note: If Adreno Profiler is having difficulty connecting to your application, try killing the ADB server with adb kill-server on the command line. Then close your application, launch Adreno Profiler, then launch your application and attempt to connect again.
