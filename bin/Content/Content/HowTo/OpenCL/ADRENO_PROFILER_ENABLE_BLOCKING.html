Adreno Profiler
ADRENO_PROFILER_ENABLE_BLOCKING Reference

Summary:

Property used to enable blocking of the execution of an application on device while Adreno Profiler establishes a connection.

Usage:

To enable blocking:



To disable blocking:



Description:

When OpenCL profiler is enabled through ADRENO_PROFILER_ENABLE_OPENCL, non-blocking is enabled by default. If blocking of the execution is needed this property needs to be set.

It's important to note that blocking execution with this method can only be accomplished on command-line applications executed through an adb shell as attempting it on APK applications will have undefined behavior. If blocking of an APK based application is needed the user will need to instrument some kind of waiting mechanism.

Note: As explained on ADRENO_PROFILER_ENABLE_WAIT_THRESHOLD in order for an application to attempt to connect to profiler there needs to be at least one API called to bootstrap the connection mechanism. This needs to be taken into account if a manual blocking instrumentation is needed in order to wait for Adreno Profiler to connect to an APK based application before starting the OpenCL code execution.
